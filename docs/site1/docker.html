What is Docker?

Docker is an open platform for developing, shipping, and running applications using containerization technology. Containers are lightweight and contain everything needed to run the software.

Key Concept: Docker allows you to package an application with all of its dependencies into a standardized unit called a container.

Installing Docker

Ubuntu/Debian

curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh

Start and Enable Docker

sudo systemctl start docker
sudo systemctl enable docker
Note: It's recommended to add your user to the docker group to run Docker without sudo:
sudo usermod -aG docker $USER

Basic Docker Commands

Pull an Image

Download a Docker image from Docker Hub

docker pull nginx:latest

List Images

View all locally stored Docker images

docker images

Run a Container

Start a new container from an image

docker run -d --name my-nginx -p 80:80 nginx:latest
  • -d - Run in detached mode (background)
  • --name - Assign a name to the container
  • -p 80:80 - Map port 80 (host) to port 80 (container)

List Running Containers

Show all running Docker containers

docker ps

List All Containers

Show all containers (running and stopped)

docker ps -a

Stop a Container

Stop a running container

docker stop my-nginx

Start a Container

Start a stopped container

docker start my-nginx

Remove a Container

Delete a stopped container

docker rm my-nginx

View Logs

View logs from a container

docker logs -f my-nginx

Use -f to follow logs in real-time

Execute Command in Container

Run commands inside a running container

docker exec -it my-nginx bash
  • -i - Interactive mode
  • -t - Allocate a pseudo-TTY

Docker Volumes

Volumes persist data outside containers

Create a Volume

docker volume create my-data

Mount Volume to Container

docker run -d --name my-app -v my-data:/app/data nginx:latest

Docker Networking

Create a Network

docker network create my-network

Connect Container to Network

docker network connect my-network my-nginx

Docker Compose

Docker Compose lets you define and run multi-container applications

Example docker-compose.yml

version: '3.8'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    restart: unless-stopped
  
  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: app
    volumes:
      - db-data:/var/lib/mysql
    restart: unless-stopped

volumes:
  db-data:

Docker Compose Commands

Start Services

docker-compose up -d

Stop Services

docker-compose down

View Logs

docker-compose logs -f

Dockerfile

A Dockerfile is a script with instructions to build a Docker image

# Use official Node.js image
FROM node:18-alpine

# Set working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm install

# Copy application code
COPY . .

# Expose port
EXPOSE 3000

# Start application
CMD ["npm", "start"]

Build Image from Dockerfile

docker build -t my-app:latest .

Useful Docker Tips

Container Resource Stats: Monitor CPU and memory usage 
docker stats
Clean Up: Remove unused containers, networks, and images 
docker system prune -a
Useful Flag: View all Docker commands 
docker --help
