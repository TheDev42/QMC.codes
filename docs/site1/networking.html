Understanding Network Basics

Networking is the foundation of self-hosting. This guide covers everything from basic IP configuration to advanced network setups.

Key Concept: Understanding TCP/IP, subnetting, and routing is essential for running self-hosted services.

IP Addressing

Check IP Address

Display all network interfaces and IPs

ip addr show

Show Only IPv4

Display IPv4 addresses only

ip -4 addr

Add IP Address

Assign IP to interface

ip addr add 192.168.1.100/24 dev eth0

Remove IP Address

Remove IP from interface

ip addr del 192.168.1.100/24 dev eth0

Network Interfaces

Enable Interface

Bring network interface up

ip link set eth0 up

Disable Interface

Bring network interface down

ip link set eth0 down

Show Interface Status

Display all interfaces

ip link show

Routing

Show Routing Table

Display kernel routing table

ip route

Add Static Route

Add new route to network

ip route add 192.168.50.0/24 via 192.168.1.1

Delete Route

Remove a route

ip route del 192.168.50.0/24

Default Gateway

Set default gateway

ip route add default via 192.168.1.1

DNS Configuration

Test DNS Lookup

Query DNS for domain

nslookup google.com

DNS Lookup with dig

Detailed DNS query

dig google.com

Flush DNS Cache

Clear DNS resolver cache

sudo systemd-resolve --flush-caches

View DNS Servers

Show current DNS configuration

cat /etc/resolv.conf

Network Testing

Ping Test

Test connectivity to host

ping -c 4 google.com

Continuous Ping

Ping continuously

ping 8.8.8.8

Trace Route

Trace packet path to destination

traceroute google.com

Network Path Analysis

Modern traceroute alternative

tracepath google.com

Test Port Connectivity

Check if port is open

nc -zv example.com 80

Port Management

List Open Ports

Show all listening ports

ss -tulpn

List Network Connections

Show all network connections

ss -tan

Check Port Usage

Find process using specific port

sudo lsof -i :80

Show Process by Port

Find PID using port

sudo netstat -tulpn | grep :80

Firewall (UFW)

Allow Port

Open port for traffic

sudo ufw allow 80/tcp

Allow Service

Allow by service name

sudo ufw allow nginx

Deny Port

Block port

sudo ufw deny 80/tcp

Show Status

List all firewall rules

sudo ufw status

Enable Firewall

Turn on UFW

sudo ufw enable

Disable Firewall

Turn off UFW

sudo ufw disable

Physical Networking Basics

Network Equipment

  • Router: Connects your network to the internet, directs traffic between networks
  • Switch: Connects devices within the same network (Layer 2)
  • Access Point: Provides wireless connectivity
  • Modem: Converts digital signals to analog for internet connection
  • Firewall: Monitors and controls incoming/outgoing network traffic
Tip: For self-hosting, use a dedicated router or a router with DD-WRT/OpenWrt firmware for more control.

Home Network Setup

Typical Setup

  1. Modem - Provided by ISP, connects to internet
  2. Router - Connects to modem, creates local network
  3. Switch - Expand wired connections
  4. Servers - Your self-hosted applications

IP Address Ranges (Private)

  • 10.0.0.0 - 10.255.255.255 (Class A)
  • 172.16.0.0 - 172.31.255.255 (Class B)
  • 192.168.0.0 - 192.168.255.255 (Class C) - Most common for home networks

Port Forwarding

Port forwarding directs external traffic to the correct internal device.

Example: Forward Port 80

Forward external port 80 to internal server

# Router configuration example
External Port: 80
Internal IP: 192.168.1.100
Internal Port: 80
Protocol: TCP
Security Note: Only forward ports that are absolutely necessary and always use firewalls!

Static IP Configuration

Ubuntu/Debian (Netplan)

network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Apply Netplan

Apply network changes

sudo netplan apply

DHCP Reservations

Reserve specific IP addresses for servers so they always get the same IP.

Best Practice: Always assign static IPs to servers running self-hosted services.

VLANs (Virtual LANs)

Segment network for better security and organization.

Create VLAN Interface

Add VLAN tag to network

ip link add link eth0 name eth0.10 type vlan id 10

Assign IP to VLAN

Configure VLAN interface

ip addr add 192.168.10.1/24 dev eth0.10

ARP Table

Show MAC addresses

Common Network Terms

  • TCP: Transmission Control Protocol - reliable, ordered delivery
  • UDP: User Datagram Protocol - faster, no guarantee
  • DNS: Domain Name System - translates names to IPs
  • DHCP: Dynamic Host Configuration Protocol - auto-assigns IPs
  • NAT: Network Address Translation - shares one IP
  • VPN: Virtual Private Network - encrypted tunnel
  • Subnet Mask: Divides IP into network and host parts
  • Gateway: Entry/exit point of network
