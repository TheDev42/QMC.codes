What is Virtualisation?

Virtualisation allows you to run multiple virtual computers (VMs) on a single physical server. Each VM runs its own operating system, completely isolated from others.

üí° Key Benefit: Virtualisation maximises hardware utilisation by running multiple VMs on a single server.

Virtual Machines vs Containers

Feature Virtual Machine Container
OS Full OS required Shares host kernel
Size GBs (several GBs) MBs (lightweight)
Startup Minutes Seconds
Isolation Full isolation Process level
Examples VMware, VirtualBox, Proxmox Docker, Podman

Proxmox VE

Proxmox Virtual Environment is an open-source server virtualisation platform based on Debian Linux.

Key Features:

  • Type 1 hypervisor (runs directly on hardware)
  • Supports both VMs and containers (LXC)
  • Web-based management interface
  • High availability clustering
  • Live migration of VMs
  • Built-in backup and restore
üí° Pro Tip: Proxmox can run on a simple Intel NUC or old desktop PC - great for home labs!

Installing Proxmox VE

System Requirements:

  • CPU: 64-bit (Intel VT-x/AMD-V required for VMs)
  • RAM: Minimum 2GB (8GB+ recommended)
  • Storage: Minimum 32GB (SSD recommended)
  • Network: Ethernet adapter

Installation Steps:

# 1. Download Proxmox VE ISO from proxmox.com # 2. Create bootable USB with ISO # 3. Boot from USB and follow installer # 4. Access web interface: https://your-ip-address:8006

Creating a VM in Proxmox

Via Web Interface:

  1. Log into Proxmox web interface
  2. Click "Create VM" button
  3. Give VM a name and select OS template
  4. Allocate CPU, RAM, and storage
  5. Configure network settings
  6. Click "Create" and then "Start"

Via Command Line:

# Create VM with ID 100 qm create 100 \ --name my-vm \ --memory 2048 \ --cores 2 \ --net0 virtio,bridge=vmbr0 \ --ostype l26 # Attach ISO qm set 100 --cdrom local:iso/ubuntu.iso # Start VM qm start 100

LXC Containers in Proxmox

LXC (Linux Containers) are lightweight containers similar to Docker but with a full Linux filesystem.

Benefits of LXC:

  • Lightweight - uses less resources than VMs
  • Fast to create and start
  • Full Linux OS with systemd
  • Can run Docker inside LXC

Create LXC Container:

# Create container from template pct create 100 local:vztmpl/ubuntu-22.04-standard_22.04-1_amd64.tar.gz # Start container pct start 100 # Access container console pct enter 100

When to Use What?

Use Virtual Machines when:

  • You need full OS isolation
  • Running Windows or different Linux distro
  • Testing different kernel versions
  • Maximum security isolation required

Use LXC Containers when:

  • You need lightweight virtualisation
  • Running multiple small services
  • Resource efficiency is critical
  • Fast deployment is needed

Use Docker when:

  • Application containerisation
  • Microservices architecture
  • Development environments
  • CI/CD pipelines
