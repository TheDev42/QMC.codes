Physical Networking Basics

Understanding IP Addresses

An IP address is a unique identifier for each device on a network. IPv4 addresses consist of 4 numbers (0-255) separated by dots.

# Example IPv4 addresses 192.168.1.1 # Private IP (home router) 10.0.0.1 # Private IP (large network) 172.16.0.1 # Private IP (medium network) 8.8.8.8 # Public IP (Google DNS)

IP Address Classes:

Class A 1.0.0.1 - 126.255.255.254 (large networks)
Class B 128.0.0.1 - 191.255.255.254 (medium networks)
Class C 192.0.0.1 - 223.255.255.254 (small networks)
Class D 224.0.0.0 - 239.255.255.255 (multicast)

Subnetting

Subnetting divides a network into smaller networks. The subnet mask determines how many hosts per network.

Common Subnet Masks:

/24 255.255.255.0 - 254 hosts
/25 255.255.255.128 - 126 hosts
/26 255.255.255.192 - 62 hosts
/27 255.255.255.224 - 30 hosts
/28 255.255.255.240 - 14 hosts
/30 255.255.255.252 - 2 hosts (point-to-point)
üí° Example: Network 192.168.1.0/24 can have 254 devices (192.168.1.1 to 192.168.1.254)

Network Routers

A router connects different networks and directs traffic between them. Your home router connects your local network to the internet.

How Routers Work:

  • Gateway: The router's IP on your local network (usually .1)
  • Routing Table: Decisions on where to send packets
  • NAT: Network Address Translation (translates local IPs to public)
  • DHCP: Automatically assigns IP addresses to devices

Network Switches

A switch connects devices within a single network. Unlike a hub, it sends data only to the intended device.

Types of Switches:

  • Unmanaged: Plug and play, no configuration needed
  • Managed: Can be configured, VLAN support, QoS
  • Smart/Web-managed: Basic management features

Managed Switch Commands (Cisco-like):

# View interface status show interface status # Configure VLAN vlan 10 name IT_DEPARTMENT # Assign port to VLAN interface GigabitEthernet0/1 switchport mode access switchport access vlan 10

UFW Firewall (Ubuntu)

UFW (Uncomplicated Firewall) makes managing iptables rules easy.

Basic Commands:

# Enable firewall sudo ufw enable # Disable firewall sudo ufw disable # Check status sudo ufw status # Allow SSH sudo ufw allow ssh # Allow HTTP/HTTPS sudo ufw allow 80/tcp sudo ufw allow 443/tcp # Allow specific port sudo ufw allow 3000/tcp # Deny incoming by default sudo ufw default deny incoming
‚ö†Ô∏è Warning: Always allow SSH (port 22) before enabling the firewall, or you may lock yourself out!

Reverse Proxy with Nginx

A reverse proxy forwards client requests to backend servers.

Nginx Config:

server { listen 80; server_name example.com; location / { proxy_pass http://localhost:3000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } }

SSL/HTTPS with Let's Encrypt

Install Certbot:

sudo apt update sudo apt install certbot python3-certbot-nginx

Get SSL Certificate:

# For Nginx sudo certbot --nginx -d example.com -d www.example.com # Auto-renewal is automatic!
